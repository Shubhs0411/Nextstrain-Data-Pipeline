## 2. Prerequisites and environment setup

### 2a. Conda environments

Two separate conda environments are used. Create them once:

**`bvbrc`** ŌĆö BV-BRC CLI + Augur + tree-building tools:

```bash
conda create -n bvbrc -c conda-forge -c bioconda \
      augur mafft iqtree2 treetime nodejs python=3.11
conda activate bvbrc
pip install bvbrc-cli        # provides p3-genome-fasta, p3-all-genomes, etc.
```

**`nextclade`** ŌĆö Nextclade CLI (only needed for the optional clade overlay, Step 6):

```bash
conda create -n nextclade -c bioconda nextclade
```

### Step 0 ŌĆö Obtain source data from BV-BRC

This step is manual (website or CLI) and only needs to be done once.

#### Option A ŌĆö BV-BRC website (recommended for large datasets)

1. Go to [https://www.bv-brc.org](https://www.bv-brc.org).
2. Search for **Influenza A H3N2** genomes.
3. Select all ŌåÆ **Export** ŌåÆ **Genome Metadata (CSV/TSV)** ŌåÆ save as `BVBRC_genome.txt`
    in `H3N2_DATA/H3N2_DATA/`.
4. For sequences: **Export** ŌåÆ **Nucleotide Sequences** ŌåÆ filter by segment ŌåÆ
    export one FASTA per segment. Name them `segment_N.fasta`, place in
    `H3N2_DATA/H3N2_DATA/`, then clean headers:
    ```bash
    for N in 1 2 3 4 5 6 7 8; do
       python scripts/clean_fasta_headers.py \
          H3N2_DATA/H3N2_DATA/segment_${N}.fasta \
          H3N2_DATA/H3N2_DATA/segment_${N}.clean.fasta
    done
    ```

#### Option B ŌĆö BV-BRC CLI

```bash
conda activate bvbrc

# Fetch all H3N2 genome metadata
p3-all-genomes --eq taxon_id,11520 > genome_metadata_H3N2.tsv

# Fetch sequences for a single genome (called by fetch_fasta_by_genome_id.py internally)
p3-genome-fasta --contig <genome_id>
```

### Step 1 ŌĆö Extract genome IDs and build Path C metadata (per segment)

**Script:** `scripts/extract_genome_ids_from_path_c.py`

**What it does:**

- Reads every FASTA header from `segment_N.clean.fasta` (the Path B input with
   strain-name headers).
- Cleans each strain name: removes outer parentheses such as `(A/...)`, strips the
   `(H3N2)` subtype suffix, and normalizes `A/Human/X ŌåÆ A/X` so names match the BV-BRC
   `Strain` column exactly.
- Maps cleaned strain names to `genome_id` using:
   1. **Primary lookup:** `BVBRC_genome.txt` ŌĆö matched by `Strain` field and `Segment`.
   2. **Fallback:** `genome_metadata_H3N2.tsv` (the `p3-all-genomes` export).
- For each successfully mapped `genome_id`, builds one Nextstrain-compatible metadata
   row with columns sourced from BV-BRC:
   `strain` (= genome_id), `strain_name`, `virus`, `date`, `region`, `country`,
   `division`, `city`, `host`, `segment`, `clade`, `subclade`, `lineage`, `accession`,
   `authors`, `paper_url` (converted from PubMed ID to full URL), `journal`,
   `isolation_source`.
- **Overlay 1:** enriches clade, host, country, and date from `NEW_genome_w_clade.txt`
   when present and non-`unassigned`. This curated file takes priority over BV-BRC.
- **Overlay 2:** merges any pre-existing Path A fields from `metadata_segmentN_h3n2.tsv`
   to stay consistent with earlier builds.

**Inputs:**
- `H3N2_DATA/H3N2_DATA/segment_N.clean.fasta`
- `H3N2_DATA/H3N2_DATA/BVBRC_genome.txt`
- `genome_metadata_H3N2.tsv`
- `NEW_genome_w_clade.txt`

**Outputs:**
- `H3N2_DATA/H3N2_DATA/genome_ids_segmentN_pathc.txt` ŌĆö one `genome_id` per line
   (~550 IDs per segment in the reference H3N2 dataset).
- `H3N2_DATA/H3N2_DATA/metadata_segmentN_pathc.tsv` ŌĆö tab-delimited Nextstrain-format
   metadata, one row per `genome_id`.

**Commands:**
```bash
# Single segment
python scripts/extract_genome_ids_from_path_c.py --segment 1

# All 8 segments manually
for N in 1 2 3 4 5 6 7 8; do
   python scripts/extract_genome_ids_from_path_c.py --segment $N
done
```

### Step 2 ŌĆö Fetch FASTA sequences by genome ID (per segment)

**Script:** `scripts/fetch_fasta_by_genome_id.py`

**What it does:**

- Reads `genome_ids_segmentN_pathc.txt`.
- For each `genome_id`, calls `p3-genome-fasta --contig <genome_id>` via parallel worker
   threads (default: 32 workers, configurable with `--workers`).
- Writes all returned contig sequences into a single FASTA output file.
- Supports `--resume`: skips any `genome_id` already present in the output file. Safe to
   restart after an interruption.

**Tip ŌĆö BV-BRC website download (much faster for large datasets):**
For 5,000+ genomes the CLI can take many hours. Instead:
1. Go to BV-BRC ŌåÆ create a **Genome List** from `genome_ids_segmentN_pathc.txt`.
2. Export ŌåÆ **Nucleotide Sequences**.
3. Save as `segment_N_h3n2_pathc.fasta` in `H3N2_DATA/H3N2_DATA/`.

**Inputs:**
- `H3N2_DATA/H3N2_DATA/genome_ids_segmentN_pathc.txt`

**Outputs:**
- `H3N2_DATA/H3N2_DATA/segment_N_h3n2_pathc.fasta` ŌĆö raw FASTA; may contain multiple
   contigs per genome (e.g., `.con.0001`, `.con.0002`).

**Commands:**
```bash
# Single segment via CLI
python scripts/fetch_fasta_by_genome_id.py 1 \
   --genome-list H3N2_DATA/H3N2_DATA/genome_ids_segment1_pathc.txt \
   --output-suffix _pathc \
   --workers 32

# Resume an interrupted fetch
python scripts/fetch_fasta_by_genome_id.py 1 \
   --genome-list H3N2_DATA/H3N2_DATA/genome_ids_segment1_pathc.txt \
   --output-suffix _pathc \
   --resume
```

### Step 3 ŌĆö Normalize FASTA headers to genome_id (per segment)

**Script:** `scripts/ensure_fasta_headers_genome_id.py`

**What it does:**

- Reads `segment_N_h3n2_pathc.fasta`.
- Extracts the `genome_id` from each contig header (BV-BRC names contigs like
   `518978.3.con.0001` ŌĆö the genome_id is the prefix before `.con.`).
- Filters to keep only sequences whose `genome_id` appears in
   `metadata_segmentN_pathc.tsv`. Sequences with no metadata row are discarded.
- **Deduplicates** on `genome_id` ŌĆö first occurrence wins. This prevents `augur align`
   from failing on duplicate sequence names.
- Rewrites every FASTA header to just the bare `genome_id`.

**Inputs:**
- `H3N2_DATA/H3N2_DATA/segment_N_h3n2_pathc.fasta`
- `H3N2_DATA/H3N2_DATA/metadata_segmentN_pathc.tsv`

**Outputs:**
- `H3N2_DATA/H3N2_DATA/segment_N_h3n2_pathc.clean.fasta` ŌĆö one sequence per
   `genome_id`; headers are plain genome_id strings only.

**Commands:**
```bash
python scripts/ensure_fasta_headers_genome_id.py --suffix _pathc --segment 1
# Repeat for --segment 2 through 8, or let Step 4 run them automatically
```

### Step 4 ŌĆö Build per-segment phylogenetic trees and Auspice JSONs

**Script:** `scripts/build_segment_pathc.py`

**What it does:**

This is the main orchestrator for per-segment tree building. If outputs from Steps 1-3
are missing for a segment, it runs them automatically before proceeding.

| Sub-step | Tool | Output |
|----------|------|--------|
| 4a | `augur align` (MAFFT) | `results/segment_N_pathc/aligned.fasta` ŌĆö multiple-sequence alignment |
| 4b | `augur tree` (IQ-TREE2) | `results/segment_N_pathc/tree.nwk` ŌĆö raw unrooted ML tree |
| 4c | `augur refine` (TreeTime) | `results/segment_N_pathc/tree-refined.nwk` ŌĆö time-resolved, rooted tree. Uses `clock_rate` and `coalescent` from `config/config.yaml`. |
| 4d | `augur traits` | `results/segment_N_pathc/traits_host.json` ŌĆö ancestral state reconstruction for `host` column |
| 4e | `augur export v2` | `auspice/h3n2_segmentN_pathc.json` ŌĆö Auspice v2 JSON combining tree, metadata, and traits |
| 4f | `node normalize_geo_resolutions.js` | Standardizes country/region geo-coordinate entries in the JSON |

After building all segments with `--all`, the script prints a summary table reporting
input header counts, mapped genome IDs, metadata rows, clean FASTA sequence counts, and
unique clade/subclade counts per segment ŌĆö useful for a quick sanity check.

**Inputs (per segment N):**
- `H3N2_DATA/H3N2_DATA/segment_N_h3n2_pathc.clean.fasta`
- `H3N2_DATA/H3N2_DATA/metadata_segmentN_pathc.tsv`
- `config/config.yaml`

**Outputs (per segment N):**
- `results/segment_N_pathc/aligned.fasta`
- `results/segment_N_pathc/tree.nwk`
- `results/segment_N_pathc/tree-refined.nwk`
- `results/segment_N_pathc/traits_host.json`
- `auspice/h3n2_segmentN_pathc.json`

**Commands:**
```bash
# Build one segment
python scripts/build_segment_pathc.py --segment 1

# Build ALL segments 1-8 in sequence (recommended)
python scripts/build_segment_pathc.py --all
```

> **Runtime note:** Each segment takes ~10-30 minutes depending on hardware and sequence
> count (~550 sequences for the Path C reference dataset). Running `--all` typically
> takes 2-4 hours total.

### Step 5 ŌĆö Enrich Auspice JSONs with metadata and colorings

**Script:** `scripts/tune_auspice_meta.py`

**What it does:**

- Reads the Auspice JSON produced in Step 4 for a given segment.
- For each **tip node** (identified by `genome_id`), injects rich metadata into
   `node_attrs`:
   - `strain`, `strain_name`, `country`, `region`, `host`, `num_date`
   - `clade`, `subclade`, `lineage`
   - `accession`, `authors`, `paper_url`, `journal`
   - `division`, `city`, `isolation_source`
- Adds **colorings** for: `country`, `region`, `host`, `clade`, `subclade`, `year`.
- Adds **filters** so users can interactively narrow tree tips in Auspice by those fields.
- Computes `meta.geo_resolutions` from a built-in coordinate table covering ~60 countries
   and 6 world regions. Only entries present in the data are included.
- Falls back to parsing year and host directly from the strain name when metadata is
   incomplete (e.g., `A/California/04/2009` ŌåÆ year `2009`).
- Also loads `NEW_genome_w_clade.txt` for a bidirectional strain-name-to-clade match
   when the primary metadata lookup misses a tip.
- **Automatic mode detection:** if the JSON filename contains `pathc_concat`, the script
   reads `metadata_pathc_concat.tsv` instead of a per-segment TSV.
- Overwrites the JSON in place.

**Inputs:**
- `auspice/h3n2_segmentN_pathc.json`
- `H3N2_DATA/H3N2_DATA/metadata_segmentN_pathc.tsv`
- `NEW_genome_w_clade.txt`
- `H3N2_output/bvbrc_enriched.tsv` (optional ŌĆö produced by `fetch_bvbrc_metadata.py`)

**Outputs:**
- `auspice/h3n2_segmentN_pathc.json` (overwritten in place)

**Commands:**

### Step 7 ŌĆö Build whole-genome concatenated tree

**Script:** `scripts/build_pathc_concat.py`

**What it does:**

Joins all eight segments into one per-isolate concatenated alignment and builds a single
whole-genome phylogenetic tree.

**Metadata merging logic (key detail):**
- Only isolates present in **all eight** segment metadata files (matched by `strain_name`)
   are included. Partial-genome isolates are excluded.
- Sequences are concatenated in fixed order: segments 1-2-3-4-5-6-7-8.
   Total alignment length Ōēł ~13 kb for H3N2.
- The tip name in the output FASTA and tree is the **HA (segment 4) `genome_id`**.
- **Demographics** (country, region, host, date, division, city, etc.): first non-empty
   value wins, sampling segments in priority order `4, 1, 2, 3, 5, 6, 7, 8` (HA first).
- **Clade/subclade:** priority order `4 (HA) ŌåÆ 6 (NA) ŌåÆ 1 ŌåÆ 2 ŌåÆ 3 ŌåÆ 5 ŌåÆ 7 ŌåÆ 8`.
   `unassigned` and empty values are always skipped.

**Tree-build pipeline sub-steps** (same tools as Step 4):

| Sub-step | Tool | Output |
|----------|------|--------|
| 7a | `augur align` (MAFFT) | `results/pathc_concat/aligned.fasta` |
| 7b | `augur tree` (IQ-TREE2) | `results/pathc_concat/tree.nwk` |
| 7c | `augur refine` (TreeTime) | `results/pathc_concat/tree-refined.nwk` |
| 7d | `augur traits` | `results/pathc_concat/traits_host.json` |
| 7e | `augur export v2` | `auspice/h3n2_pathc_concat.json` |
| 7f | `node normalize_geo_resolutions.js` | Standardizes coordinate entries |

**Inputs:**
- `H3N2_DATA/H3N2_DATA/segment_{1-8}_h3n2_pathc.clean.fasta` (all 8 must exist)
- `H3N2_DATA/H3N2_DATA/metadata_segment{1-8}_pathc.tsv` (all 8 must exist)
- `config/config.yaml`

**Outputs:**
- `H3N2_DATA/H3N2_DATA/h3n2_pathc_concat.clean.fasta` ŌĆö concatenated genome FASTA
   (one entry per isolate; HA genome_id as header).
- `H3N2_DATA/H3N2_DATA/metadata_pathc_concat.tsv` ŌĆö merged metadata with union of all
   segment columns (one row per isolate).
- `results/pathc_concat/aligned.fasta`, `tree.nwk`, `tree-refined.nwk`, `traits_host.json`
- `auspice/h3n2_pathc_concat.json`

**Command:**
```bash
python scripts/build_pathc_concat.py
```

> **Windows note:** `mafft` and `iqtree2` are not available on native Windows.
> Run this in **WSL** with `conda activate bvbrc`. The FASTA and TSV generation
> (pure Python) work on Windows; only the tree-build sub-steps require Linux.

## 9. Troubleshooting

| Problem | Likely cause | Fix |
|---------|-------------|-----|
| `WinError 2` or `FileNotFoundError: mafft` | `mafft`/`iqtree2` not on native Windows PATH | Run all tree-build commands inside **WSL** with `conda activate bvbrc` |
| `augur align` crashes with out-of-memory | Too many sequences at once | Set `subsample_max_sequences: 5000` in `config/config.yaml` |
| `No common strain_name values` in concat build | A segment metadata TSV is missing or lacks `strain_name` column | Re-run Step 1 for all 8 segments; verify each TSV has the `strain_name` column |
| Nextclade returns all `unassigned` clades | Wrong dataset or truncated FASTA input | Use `segment_N_h3n2_pathc.clean.fasta` (not the raw `.fasta`); confirm dataset: `flu_h3n2_ha` for segment 4, `flu_h3n2_na` for segment 6 |
| `extract_genome_ids_from_path_c.py` maps 0 IDs | `BVBRC_genome.txt` column names differ | Open the TSV and verify that `Strain` and `Segment` columns exist with those exact headers |
| All tips in JSON show empty `clade` | `NEW_genome_w_clade.txt` missing or column 4 is blank | Verify the file path and that column index 4 (0-based, tab-delimited) contains clade strings |
| Concat tree build runs for a very long time | Normal ŌĆö IQ-TREE2 on ~13 kb ├Ś ~500 sequences is computationally intensive | Let it run; typically completes in 30-90 minutes on a 4-core machine |
| `p3-genome-fasta` returns empty for some IDs | BV-BRC genome is retired or inaccessible | Those IDs are skipped automatically; use `--resume` to continue and check affected IDs on BV-BRC |
